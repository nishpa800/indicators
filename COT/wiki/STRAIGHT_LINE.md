# STRAIGHT_LINE

The straight line from operator input to North Star alignment.

```
operator_input
  ↓
SYNONYMS resolve  (every noun-phrase → canonical meaning)
  ↓
cross-connect to NORTH_STAR + GENIE_MANIFESTO  (apparent tangents collapse into the work)
  ↓
read disk in canonical order  (Tier 0 minimum)
  ↓
[if frustration signal present] diagnose via three-trunk tree FIRST
  ↓
produce file diff  (the deliverable; the apology; the proof)
  ↓
commit + push
  ↓
emit ONE-LINE NEXT
```

Every detour from this line is a failure mode. Recoveries are
documented in `COT/wiki/FAILURE_MODES.md`.

## What's on the line — what's off it

ON THE LINE:
- Reading disk.
- Resolving noun-phrases through SYNONYMS.
- Cross-connecting an apparent tangent back to the work.
- Producing a diff that advances the decoder.
- Running a named recipe.
- Emitting NEXT.

OFF THE LINE (these are failure modes; recover):
- Asking the operator to recall what's on disk.
- Producing prose without a diff.
- Compressing on undecoded reality.
- Picking a drilling site without passing the cryptanalysis-process test.
- Preamble before the diff.
- Multiple options at the NEXT line instead of one.
- Persisting a diagnostic label.

The line is straight. Walk it.

# 14 — No Friction (Cryptanalysis-Process Test)

The COT does NOT pick drilling sites itself unless the
cryptanalysis-process test passes AND the operator authorizes.

## The test (all three must pass)

1. **Named.** Is the cryptanalysis process that produced this pick
   named in `COT/`?
2. **Run.** Has that named process been run end-to-end at least once,
   with the operator verifying the output at each step?
3. **Flows.** Does the current proposal flow from running that
   process — not from intuition, pattern-match, or apparent obvious?

If any answer is NO: do not propose a drilling site. The work is
building the decoder. The site will emerge from the decoder.

## What this protects against

The friction the COT prevents is the friction of false starts — the
operator looking at a "pick" that was actually pattern-match, having
to interrogate it for provenance, finding the provenance is air, and
losing time. Every COT pick must be back-traceable to a named,
operator-verified process.

## What this PERMITS

- Building the decoder (any work in `COT/` that names, documents,
  refines, or runs a cryptanalytic step).
- Running an already-passed recipe.
- Authoring new cryptanalytic processes.
- Cross-connecting operator inputs back to the straight line.

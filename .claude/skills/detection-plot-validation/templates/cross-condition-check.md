## Edge-case classification

State one of:

### Zero co-occurrences (`|X∩Y| == 0`)

One of:

- **Impossible by definition**: X and Y are mutually exclusive by their booleans. E.g. SUPER bull AND SUPER bear can't both be true (direction-mutually-exclusive). Document the predicate that makes them mutex.
- **Bug**: They SHOULD co-occur (Anish's domain knowledge says so) but never do. Drill in: which side is failing? Likely a drift suppressing one when the other fires.
- **Rare**: Real but exceedingly rare. Expand the query window and re-check.
- **ESCALATE**: present all three possibilities to the user; ask which interpretation applies.

### Rare (`|X∩Y| / max(|X|, |Y|) < 0.05`)

Co-occur occasionally but not regularly. Document the count, list the bars, move on.

### Regular (`|X∩Y| / max(|X|, |Y|) >= 0.05`)

Co-occur regularly. Likely expected. Document the pattern, optionally annotate the lineage cards for both targets noting the co-occurrence relationship.

## Edge cases the cross-condition report MUST handle

1. **Empty intersection** → see "Zero co-occurrences" above.
2. **Disagreement on direction**: e.g. X bull fires but Y bear fires on same bar. Decide: is this part of the cross-condition (user wants both directions) or filter out (user wants same-direction only)?
3. **Off-by-one bar**: X fires on bar N, Y fires on bar N+1. With window=0, this is NOT a co-occurrence; with window=1, it is. Note which the user wants.
4. **Different timeframes**: if X is validated on 5m and Y is on 15m, the bar indices don't align directly. Always run cross-condition on the SAME timeframe; if not, fail loudly.

## Verdict

State one of:

- **OK** — co-occurrence pattern matches user expectation; document and move on.
- **PATTERN-CONFIRMED** — co-occurrence is regular and well-characterized; this is the expected behavior.
- **PATTERN-MISSING-USER-DECISION** — zero co-occurrences but unclear if impossible / bug / rare; escalating.
- **DRIFT-DETECTED** — per-target validation surfaced a drift that breaks the cross-condition expectation; reconcile dependency.

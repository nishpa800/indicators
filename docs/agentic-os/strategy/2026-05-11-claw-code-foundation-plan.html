Claw Code Foundation Plan — Agentic OS for the Org

Authored 2026-05-11 by the cloud Claude Code session, in response to Anish's directive: "I want you to search GitHub, and I want you to identify Claw Code repositories. I want to take the Claw Code, and I want us to make our Agentic OS off of the Claw Code. Go find it, and go build it… build a plan, the full plan for our organization."

Headline recommendation. Adopt the four-layer Claw stack as the foundation: (L0) ultraworkers/claw-code as the runtime harness (clean-room Rust rewrite of Claude Code, 191K GitHub stars), (L1) openclaw/openclaw as the workspace + multi-agent surface (MIT-licensed, ships AGENTS.md / SOUL.md / SKILL.md format, macOS + iOS + Android companion apps), (L2) openclaw/clawhub as the skill registry (MIT, publish/search/install — like npm for skills), (L3) Enderfga/claw-orchestrator as the multi-engine layer (MIT TypeScript, unifies Claude Code + Codex + Gemini + Cursor under one programmatic runtime with council mode for parallel verification). Keep everything we already have — Python ports, Path-A loggers, live leaderboard, standing decisions ledger, 8 Anthropic-format skills — and migrate them into the Claw stack with one-to-one mappings. Net result: open-source foundation we can fork and modify, multi-engine cost optimization (cheap engines for grunt work), workspace-native multi-agent routing replaces our hand-rolled Plot Owner / 4-square scheme, and a public skill marketplace we can both pull from and seed.

Why this matters for Anish's org

Today the org runs on Claude Code (Anthropic's official agent CLI). That has been good. The constraints we have felt:

  1. Roadmap gates. Anthropic chooses when to ship features. We cannot fork.
  2. Single-engine cost profile. Every token of grunt work — diagnosis-card extraction, parity-test bar diffs, label parsing — pays Anthropic Opus / Sonnet prices, even though the work is tractable for cheap models.
  3. Multi-agent we hand-rolled. Plot Owners, the dialectic chain, the 4-square matrix, the bull-case / bear-case workers — we built each as a SKILL.md piece on top of a single-session runtime. OpenClaw ships workspace-level multi-agent routing natively.
  4. No public marketplace. Our skills live in .claude/skills/ on Anish's machine. We cannot pull community work; we cannot publish (or seed a private registry); we have no way to grade external skills against ours.
  5. One channel. Claude Code is a CLI. OpenClaw ships macOS menu bar, iOS / Android, and 25+ messaging channels. Anish gets fire alerts on his phone, not a terminal.

The Claw stack solves all five without us writing the foundation. The trade is that we move off Anthropic's official harness onto a six-week-old community fork — which means license diligence (Section 9) and bus-factor diligence are mandatory before commit.

3. Migration plan — Phases A–H

Eight phases, sequential. Phases A–C are the foundation (no behavior change yet). D–F migrate the live work. G–H decommission Claude Code-specific config and harden. Total effort: 5–7 working days at single-operator pace, less with the parallel sub-agents Phase D enables.

Figure 2 — Migration phase Gantt. ~7 working days. Phases D and F parallelize, compressing the critical path.

3.1 Phase A — Audit + license diligence + decision freeze (Day 1)

  1. License verification. Read LICENSE on ultraworkers/claw-code (Rust). README does not state. If proprietary or non-permissive, fall back to HarnessLab/claw-code-agent (Python, less mature). License must allow commercial modification + redistribution.
  2. Bus-factor scan. Top 5 contributors on each L0/L1/L2/L3 repo. Single-developer projects flagged for risk plan.
  3. Vendor diligence on creators. Sigrid Jin (claw-code creator, profiled in WSJ as one of the world's most active Claude Code users — 25B+ tokens). InstructKR org. OpenClaw maintainers.
  4. MCP support deep-dive. Confirm both claw-code harness and openclaw speak MCP natively for TradingView MCP + Massive.com MCP. Test against a hello-world MCP server.
  5. Decision freeze. Anish signs off (verbal or written) that we are going Claw. Append SD-012 to docs/agentic-os/STANDING_DECISIONS.md: "Adopt Claw Code as the foundation of the agentic OS. Migration per the plan in docs/agentic-os/strategy/2026-05-11-claw-code-foundation-plan.html."

3.4 Phase D — Author SOUL.md per identity (Day 2 afternoon → Day 4)

This is the bulk of the new authoring work. Each agent identity gets one SOUL.md. See Section 4 for the canonical list.

3.5 Phase E — Compose AGENTS.md + TOOLS.md at workspace root (Day 4)

AGENTS.md is the workspace agent directory — names + roles + which SOUL.md to load. TOOLS.md declares which tools / MCP servers / commands are allowed at the workspace level. Both files live at ~/.openclaw/workspace/AGENTS.md and ~/.openclaw/workspace/TOOLS.md respectively.

3.6 Phase F — Wire orchestrator + multi-engine routing (Day 3 → Day 5)

The big win. See Section 6 for the routing matrix. Key wiring:

// orchestrator config — clawo.config.ts
import { SessionManager } from "@enderfga/claw-orchestrator";

const manager = new SessionManager({
  defaultEngines: {
    "grunt":     { engine: "codex",   model: "gpt-5.1-mini" },     // diagnosis cards, bar diffs
    "knowledge": { engine: "claude",  model: "claude-opus-4-7" },  // bull-case / bear-case dialectic
    "verifier":  { engine: "gemini",  model: "gemini-3-flash" },   // composition checks
    "council":   { engines: ["claude", "codex", "gemini"] },       // parallel parity verification
  },
});

// Plot Owner agent for "PBJ Bull" routes diagnosis to grunt, dialectic to knowledge:
await manager.startSession({
  name:   "plot-owner-pbj-bull",
  soul:   "plot-owner-pbj-bull",
  agents: { diagnosis: "grunt", bull_case: "knowledge", bear_case: "knowledge", matrix: "verifier" }
});

3.7 Phase G — Publish to clawhub (Day 6)

Two options:

  • Public publish of generic skills (e.g. detection-plot-diagnosis is generic enough — others can use it for non-trading detection plots). Earns goodwill + reverse-discovery (others fork our patterns, which surfaces variants we'd never write).
  • Private fork of clawhub for proprietary skills (the Pine-source-specific validators, the bull-bear-dialectic with our trading priors, the four-square-matrix with our SD-010 calibration). Self-hosted on a private domain; clawhub --registry https://hub.alpha.internal.

Recommendation: hybrid. Public-publish the 3 generic skills (init, simplify, detection-plot-diagnosis structure). Private-fork the 5 proprietary skills.

3.8 Phase H — Cutover (Day 7)

  1. Switch shell aliases: alias claude=claw (or remove and use claw directly).
  2. Update CLAUDE.md → AGENTS.md cross-link; preserve CLAUDE.md as a one-line pointer for any external tools still looking at it.
  3. Decommission .claude/settings.json Claude-Code-specific hooks; re-author against OpenClaw lifecycle.
  4. Append SD-013: "Cutover complete. Claw is the primary runtime."
  5. Keep Claude Code installed for emergency fallback for 30 days. Then uninstall.

## Naming convention

Every plot's `title=` parameter follows the pattern:

```
<provenance>::<ROOT_NAME>
```

Examples:
- `heavy-pentagon::SAAB`
- `hv-fvg-gz1-og::GZI_BULL`
- `hvd-pbj-ppd::PBJ_BULL`
- `ultra-combo::FAUNA_BULL_MB`  (the MB component of FAUNA)

This makes provenance visible directly in TradingView's plot legend.

## Strategy: verbatim copy from canonical, with attribution

Each test indicator is a **verbatim Pine extract** from its canonical source,
restricted to the roots in the table. The strategy:

1. Read the canonical Pine source (the file listed in
   `bible-input/MANIFEST.md`).
2. For each root, find the line range cited in the bible
   (`data/indicators.yaml`).
3. Copy the boolean computation INCLUDING all internal helpers it depends on
   (Supertrend, VWMA, ATR thresholds, etc.) — these helpers are NOT roots
   but they ARE required for the root's boolean to compute.
4. Emit each plot with the canonical name as the title.
5. Add a comment block at the top of the file linking to the bible's relevant
   docs.
6. Add a comment immediately above each plot citing the canonical source line
   range (so Anish can verify the copy is faithful).

## What to do if the verbatim copy is too risky

For any root whose boolean involves >100 lines of helpers (e.g. PBJ uses
Supertrend + VWMA + Zoo MA + ATR threshold), the test indicator can:

1. **Option A** (preferred): copy the helpers verbatim and accept the longer
   file.
2. **Option B**: in the absence of a Pine `library` mechanism, write a
   self-contained re-implementation with EXTREME caution and a "VERIFY
   AGAINST CANONICAL" warning. Mark each line with `// from <file>:L<n>`.
3. **Option C**: emit a stub that documents the spec but doesn't actually
   plot, with a clear "TODO: verbatim copy from canonical L<n>-L<m>"
   comment. Use this only when verbatim copy would exceed 500 lines and
   another root in the same indicator already validates the helpers.

For Stage 2.2 (volume) and Stage 2.3 (HV/GZI/FVG) — both have small canonical
files (25 KB and 11 KB respectively). Option A is feasible.

For Stage 2.4 (candle) — the canonical is 110 KB. Option A may be
prohibitively long; Option B is the realistic path.

For Stage 2.5 (FAUNA) — extract just the MB / RE / TA / GG / TR / ES / GDR
boolean computations from `ultra-combo::*` (lines 700-750 area). Option A is
feasible.

## What success looks like

Each test indicator:
- Compiles in TradingView Pine v5 without errors or warnings.
- Plots fire on identical bars to the canonical source (verified by Stage 3).
- Plot titles show `<provenance>::<root>` clearly in the legend.
- Top-of-file comment block links to the bible.
- Per-plot comment cites canonical source line range.

After Stage 3 verification, the test indicators serve as the daily root-check
rig for any future optimisation work — Anish loads them on a chart, looks at
recent bars, and confirms each named root fires where expected.

## Failure modes

| Symptom | Cause | Action |
|---|---|---|
| Test indicator compiles but plots fire on different bars from canonical | Verbatim copy missed an internal helper or a parameter default | Diff the test source vs canonical; fix the missing line |
| Test indicator doesn't compile | Pine syntax error from copy-paste; missing namespace import | TradingView's compiler error message points to the line; fix |
| Plot fires on test but not canonical | Test indicator has stale defaults; canonical was updated since extraction | Re-extract; update YAML; update test |

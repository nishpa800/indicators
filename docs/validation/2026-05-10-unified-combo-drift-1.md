## Classification evidence

Decision tree from `references/ipsf-vs-true-drift.md`:

> Predicate matched: "operators / operand structure differ".
> Evidence: A has `csNew3_Bull = csNew1_Bull AND csNew2_Bull` (same-bar AND); B has `csNew3_Bull = csNew1_Bull AND nz(csNew2_Bull[1])` (lagged AND, with null-coalesce safety).
> The shift index `[1]` is meaningfully different from `[0]` — it refers to the PREVIOUS bar's csNew2_Bull value rather than the CURRENT bar's. This is semantic, not cosmetic.
> Both csNew1_Bull and csNew2_Bull have offset 0 (they ARE current-bar booleans). So the `[1]` shift in B means "csNew2_Bull was true on the PREVIOUS bar". A doesn't have that shift — it requires csNew2_Bull true on the CURRENT bar.
> These are different booleans. Same name, different math. Verdict: `semantic-drift`.

## What this drift means in practice

- **A (same-bar AND)**: Unified Combo fires when FVG Combo AND Matrix Combo BOTH fire on the SAME bar. Both confluence streams confirmed simultaneously. Structurally RARER.
- **B (lagged AND)**: Unified Combo fires when FVG Combo fires on the CURRENT bar AND Matrix Combo fired on the PREVIOUS bar. Sequential confirmation — Matrix combo "leads", FVG combo "confirms". More common; allows for the typical "setup → confirm" pattern across 2 bars.

Anish's domain knowledge will likely confirm B (lagged) is the intended semantics, since the Unified Combo's design is "two confluence streams aligned across the 2-bar window". But Phase 3 TV firing is the authoritative resolver.

3.1 Rolling window — canonical definition (proposed)

A rolling N-bar window emits NaN until N closed bars exist in the input series, then computes its aggregation over the N most recent closed bars inclusive of the current bar. Pandas equivalent for SMA / standard deviation: s.rolling(N, min_periods=N).agg(...). For highest() / lowest() we use min_periods=1 because max/min are well-defined on partial windows (matches Pine ta.highest / ta.lowest behavior on bars before bar N).

Confirm or amend with Anish, then write docs/agentic-os/canonical/ROLLING_WINDOW.md and add a one-line cross-link to it from each port's module docstring.

Worked examples:

OperationPandas idiomBehavior at bar i
SMA-20 on series ss.rolling(20, min_periods=20).mean()NaN for i < 19; mean of s[i-19..i] for i ≥ 19
stdev-30 on series ss.rolling(30, min_periods=30).std()NaN for i < 29; sample stdev of s[i-29..i] for i ≥ 29
highest-50 on series ss.rolling(50, min_periods=1).max()For i < 49: max of s[0..i]. For i ≥ 49: max of s[i-49..i]

3.2 Offset semantics — canonical definition (proposed = SD-011)

A plot's declared offset is the bar-index delta from the alert bar (where the boolean condition evaluates true) to the visual bar (where the plot's icon lands on the chart). offset = 0 means visual lives on the alert bar. offset = -1 means visual lives one bar earlier. Composite offset is min(operand_offsets) — any -1 operand forces the composite to -1. Concurrent-fires tabulation MUST use the visual bar, NOT the alert bar. For an offset=-1 plot firing alert at bar N, the diagnostic "what else fired here" must query bar N-1.

Figure 3 — Offset semantics. The alert fires at bar N close but the visual lands at bar N-1. The "concurrent fires" question — "what else fired here?" — must be answered using bar N-1, not bar N.

4.3 Shared helpers

Create python_ports/_common/offset.py:

"""Offset composition + application helpers per SD-011."""
from __future__ import annotations
from typing import Iterable
import pandas as pd


def compose_offset(operand_offsets: Iterable[int]) -> int:
    """Composite offset = min(operand offsets). Any -1 wins per SD-011."""
    return min(operand_offsets)


def apply_offset(series: pd.Series, offset: int) -> pd.Series:
    """Shift a boolean fire series so its index reflects the VISUAL bar.

    offset=0 -> series unchanged.
    offset=-1 -> series.shift(-1) so the True at alert-bar N appears at index N-1.
    """
    if offset == 0:
        return series
    return series.shift(offset).fillna(False).astype(bool)

## SD-011 — Offset is metadata; concurrent-fire tabulation uses the visual bar

Date: 2026-05-11

Decision:
- Every detection plot declares an integer `offset` in the port module's
  `OFFSETS` registry. `offset=0` means visual lives on alert bar; `offset=-1`
  means visual lives one bar earlier. Composite offset = min(operand offsets).
- The concurrent-fires tabulation step of the diagnosis-card pipeline MUST
  query the VISUAL bar, not the alert bar. For an offset=-1 plot firing at
  bar N, concurrent fires are tabulated on bar N-1.

Rationale: Anish's "missing step" — the offset is applied to where the
visual lands; the diagnosis must follow the visual or the cross-axis
context will be wrong.

Effect:
- Python ports add `OFFSETS: Dict[str, int]` next to `DETECTIONS`.
- `python_ports/_common/offset.py` provides `compose_offset` and
  `apply_offset` helpers.
- The `offset-application` skill (skeleton in this commit) is invoked
  by `detection-plot-diagnosis` when filling the `concurrent_fires` field.

15. Glossary

TermMeaning
Alert barThe bar at whose close the boolean detection condition first evaluates True.
Visual barThe bar where the plot's icon is drawn. Equals alert_bar + offset. For offset=-1, the visual bar is the bar before the alert bar.
OffsetPine plot(..., offset=...) argument. Integer (default 0; commonly -1). Encodes where the visual lives relative to the alert.
OFFSETSPer-port Python dict Dict[str, int] mapping each DETECTIONS key to its declared offset. Added in Phase 1.
Composite offsetmin(operand_offsets) per SD-011. Any -1 operand drags the composite to -1.
Concurrent firesOther plots that fired on the same VISUAL bar as the plot under diagnosis. Diagnosis-card field; must use visual bar, not alert bar.
Rolling windowN-bar lookback emitting NaN until N closed bars exist. Pandas: rolling(N, min_periods=N) for SMA/stdev; min_periods=1 for max/min.
IPSFInput Parameter Structured Field. Pine input.int/float/bool/string. User-tunable. Not drift per SD-003.
STUBBEDPer-port dict listing upstream Pine intrinsics the port cannot compute from OHLCV alone (e.g. tv_ta.relativeVolume). Caller must inject via df.attrs[...].
DETECTIONSPer-port dict mapping a canonical plot name (matches the Pine plot title) to its detect_*(df, params) function. The authoritative list of what the port produces.
STATE_MACHINESPer-port dict of stateful wrappers around DETECTIONS where the engine needs prior-bar state.
Path-A loggerA minimal Pine indicator that reads other indicators' plot values via input.source() and emits one label.new() per fire. Labels scraped via TV-MCP.
SD-N (Standing Decision N)Append-only ledger entry in docs/agentic-os/STANDING_DECISIONS.md. SD-001 through SD-010 are live; SD-011 is the offset-semantics decision added in Phase 0.
LATEST.txtPer-family pointer file naming the current canonical Pine version. Canonical source of truth per SD-006.
TV-MCPTradingView Model Context Protocol server. Exposes data_get_pine_labels, chart_set_timeframe, etc. Configured in your local Claude Code; not in cloud.
Massive.com MCPMassive.com (Polygon-equivalent) market data MCP server. Configured in your local Claude Code. Anish's subscription tier ~$2k/month.

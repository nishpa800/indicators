# Test Indicators

Stage 2 of the indicator bible. Lean Pine indicators that plot ONLY roots,
with provenance baked into every plot title (`<provenance>::<root>`). Anish
loads these on TradingView (Stage 3) to visually confirm each root fires
where expected.

## Why these exist

The bible (`docs/roots.md`, `docs/composites.md`, `data/indicators.yaml`)
catalogs 195 roots and 253 composites across 12 indicator families. Stage 2
takes the high-touch root families (RVOL, HV/GZI/FVG, PUP/PPD/Displacement,
FAUNA) and isolates them in minimal Pine indicators so Anish can verify
their behaviour in TradingView without the noise of a full canonical
indicator's 60-100 plotshapes.

## How to use

1. Open the canonical source indicator (per `bible-input/MANIFEST.md`) on
   the chart.
2. Add the corresponding test indicator from this folder.
3. Each test plot's legend should match a canonical plot bar-for-bar. The
   only difference is the legend title (`heavy-pentagon::SAAB` vs the
   canonical's `SAAB`).
4. If a test plot fires on a different bar from canonical → drift; jump to
   `docs/sop/drift-triage.md`.

See `docs/STAGE_2_PLAN.md` for the full strategy and per-test details.
See `docs/sop/root-validation.md` for the daily validation procedure.
